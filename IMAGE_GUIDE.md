# Гайд по добавлению изображений

## Быстрый старт

### 1. Добавьте ваши изображения

Поместите изображения в папку `public/examples/`:

```
public/examples/
├── styling-example.jpg        # Стилизация фото
├── weather-example.jpg        # Замена погоды
├── id-photo-example.jpg       # Фото на документы
├── retouch-example.jpg        # Ретушь
├── outfit-example.jpg         # Примерка одежды
└── ocr-example.jpg            # OCR пример
```

### 2. Раскомментируйте код в компонентах

#### В файле `src/components/NanaBananaShowcase.jsx`

Найдите строку ~60 и раскомментируйте:

```jsx
// БЫЛО (закомментировано):
{/*
<img
  src={`/examples/${item.imagePlaceholder}`}
  alt={item.title}
  className="w-full h-full object-cover group-hover:scale-110 transition-transform duration-500"
/>
*/}

// СТАЛО (раскомментировано):
<img
  src={`/examples/${item.imagePlaceholder}`}
  alt={item.title}
  className="w-full h-full object-cover group-hover:scale-110 transition-transform duration-500"
/>
```

#### В файле `src/components/OCRShowcase.jsx`

**Шаг 1:** Найдите строку ~35 и раскомментируйте изображение:

```jsx
<img
  src="/examples/ocr-example.jpg"
  alt="OCR Example"
  className="w-full h-full object-cover"
/>
```

**Шаг 2:** Найдите строку ~55 и добавьте пример распознанного текста:

```jsx
// Замените placeholder на реальный текст из вашего изображения:
<p>
  Ваш распознанный текст здесь<br />
  Может быть на нескольких строках
</p>
```

### 3. Готово!

Запустите `npm run dev` и проверьте результат.

## Рекомендации по изображениям

### Размеры и форматы

| Изображение | Размер | Формат | Макс. размер файла |
|-------------|---------|--------|-------------------|
| Нана Банана примеры | 800x600px | JPG/WebP | 200-300 KB |
| OCR пример | 800x600px | JPG/PNG | 150-200 KB |

### Качество изображений

- **Формат:** WebP (лучшее сжатие) или JPG (широкая поддержка)
- **Качество:** 80-85% (оптимально для веба)
- **Разрешение:** 72-96 DPI

### Что показывать на примерах

#### Стилизация (styling-example.jpg)
- До/После стилизации
- Желательно показать превращение в стиль Гибли или Дисней
- Четкое, качественное изображение

#### Погода (weather-example.jpg)
- Фото с измененной погодой
- Можно показать split-screen: солнце ↔ дождь
- Естественный результат

#### Фото на документы (id-photo-example.jpg)
- Профессиональное портретное фото
- Нейтральный фон
- Соответствие стандартам документов

#### Ретушь (retouch-example.jpg)
- До/После ретуши
- Естественный результат (не переборщить)
- Хорошее освещение

#### Примерка одежды (outfit-example.jpg)
- Человек в разной одежде
- Реалистичная примерка
- Четкие детали одежды

#### OCR (ocr-example.jpg)
- Документ или изображение с четким текстом
- Может быть чек, визитка, рукописный текст
- Текст должен быть разборчивым

## Инструменты для обработки изображений

### Онлайн

- **Squoosh** (https://squoosh.app/) - сжатие и конвертация
- **TinyPNG** (https://tinypng.com/) - оптимизация PNG/JPG
- **CloudConvert** (https://cloudconvert.com/) - конвертация форматов

### Десктоп приложения

- **ImageOptim** (macOS) - автоматическая оптимизация
- **GIMP** - бесплатный редактор изображений
- **Photoshop** - профессиональная обработка

### Командная строка

```bash
# Конвертация в WebP
cwebp -q 80 input.jpg -o output.webp

# Изменение размера с ImageMagick
convert input.jpg -resize 800x600 output.jpg
```

## Альтернатива: использование placeholder сервисов

Если у вас пока нет готовых изображений, можно использовать временные placeholder:

```jsx
// В компонентах замените на:
<img
  src={`https://placehold.co/800x600/png?text=${encodeURIComponent(item.title)}`}
  alt={item.title}
  className="w-full h-full object-cover"
/>
```

## Проверка результата

После добавления изображений проверьте:

1. ✅ Изображения загружаются корректно
2. ✅ Нет ошибок в консоли браузера
3. ✅ Изображения адаптивны на мобильных устройствах
4. ✅ Время загрузки страницы приемлемое (< 3 сек)
5. ✅ Изображения не искажаются при разных размерах экрана

## Troubleshooting

### Изображения не отображаются

1. Проверьте пути к файлам
2. Убедитесь что файлы находятся в `public/examples/`
3. Проверьте имена файлов (они чувствительны к регистру)
4. Перезапустите dev-сервер

### Изображения слишком большие

1. Используйте инструменты сжатия
2. Конвертируйте в WebP формат
3. Уменьшите разрешение до рекомендуемого

### Изображения загружаются медленно

1. Оптимизируйте размер файлов (< 300 KB)
2. Используйте lazy loading (уже реализовано)
3. Рассмотрите использование CDN для продакшена
